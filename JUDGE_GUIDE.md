## Executive Summary

**MAN OF STEEL** is an award-caliber, AI-powered industrial maintenance intelligence platform designed for steel manufacturing plants. It combines a **multi-agent AI system** (LangGraph), **machine learning predictions** (XGBoost-style), **semantic knowledge retrieval** (RAG), and a **stunning HUD-inspired UI** to predict equipment failures, diagnose root causes, and prevent costly downtime.

> **Tagline:** *Predict. Explain. Prevent.*

---

## What Problem Does It Solve

### The Real-World Problem
Industrial equipment failure costs the steel industry **$50B+ annually** in unplanned downtime. Maintenance teams struggle with:
- **Data overload** — Thousands of sensor readings with no actionable insights
- **Tribal knowledge loss** — Expertise locked in PDFs and manuals, not searchable
- **Reactive maintenance** — Fixing things after they break instead of predicting failures
- **Siloed systems** — Separate tools for monitoring, maintenance, and documentation

### Our Solution
MAN OF STEEL unifies **predictive analytics**, **AI diagnostics**, **knowledge management**, and **intelligent reporting** into one seamless, beautiful interface.

**Purpose:** Multi-agent AI conversation interface for maintenance intelligence.

**What judges see:**

- **Chat Messages** — User messages right-aligned, agent responses in HUD panels
- **Rich Markdown Rendering** — Bold, headers, bullet lists, colored Observation/Evidence/Risk/Action labels
- **Citation Footers** — Each agent response shows document citations with similarity scores
- **Agent Pipeline Sidebar** — Visual indicator of which agents were invoked (Planner, Knowledge, Prediction, Decision)
- **Latest Decision Panel** — Risk level, failure probability, root cause, business impact
- **Suggested Queries** — 5 pre-built questions with hover animations
- **Loading State** — "Agents investigating..." with spinner

**The 4-Agent Pipeline:**

1. **Planner** → Analyzes query intent (diagnosis, prediction, procedure, what-if)
2. **Knowledge** → Semantic search across 8 documents with cosine similarity
3. **Prediction** → XGBoost-style failure probability + RUL estimation
4. **Decision** → LLM or rule-based synthesis into root cause, risk, actions, business impact

**Key Differentiator:** When no OpenRouter key is set, the system seamlessly falls back to rule-based intelligence — the app never breaks.

## Technical Architecture Deep Dive

### Frontend Architecture

```
Next.js 16 App Router
├── Server Components (page.tsx wrappers)
├── Client Components (all feature components)
├── API Routes (REST endpoints)
├── Framer Motion (animations)
├── Tailwind CSS v4 (styling)
└── shadcn/ui primitives (Card, Button, Badge)
```

### AI Agent Architecture

```
┌─────────────────────────────────────────────────────┐
│                   LangGraph State                    │
│  query, assetId, citations, prediction, rootCause,  │
│  riskLevel, recommendedActions, businessImpact       │
└─────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────┐
│   Planner Node  │  ← Intent classification
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Knowledge Node │  ← RAG: semantic + keyword search
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Prediction Node │  ← ML: sigmoid(logits) + RUL
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Decision Node  │  ← LLM or rule-based synthesis
└────────┬────────┘
         │
         ▼
    Response + Citations
```

### ML Prediction Engine

- **Model:** XGBoost-style logistic regression (implemented in TypeScript)
- **Features:** Temperature, Pressure, RPM, Vibration, Operating Hours
- **Output:** Failure probability (sigmoid), Remaining Useful Life (RUL), risk level
- **What-If:** Perturb any feature → recalculate probability delta
- **Training:** Python scripts using scikit-learn + XGBoost (optional)

### RAG (Retrieval Augmented Generation)

- **Embeddings:** OpenRouter API (fallback to local hash-based embedder)
- **Search:** Cosine similarity on document chunks
- **Chunking:** 800-char chunks with 100-char overlap
- **Threshold:** 0.72 minimum similarity score
- **Max Citations:** 6 per query

### Data Layer

- **Default:** In-memory data store with synthetic generation
- **Optional:** Supabase PostgreSQL with pgvector for production
- **Seed Data:** 10 assets, 5 types, 72h sensor readings, alerts, maintenance records

## FAQ

### Is this just a ChatGPT wrapper?
No. The AI Copilot uses a **LangGraph pipeline** with four specialized agents: Planner (routes queries), Knowledge (RAG search), Prediction (ML model), Decision (synthesizes). This is a multi-agent system, not a single LLM call.

### Does the app require payment?
No. The app is fully functional with zero API keys, zero paid services, and zero configuration.

### Is the ML model real?
Yes. The prediction engine uses XGBoost-style coefficients loaded in TypeScript. Python training scripts are included to retrain the model on real data.

### Can I deploy this?
Yes. Deploy to Vercel for free with `npx vercel`. Works with zero configuration.

### What data does the app use?
By default, the app generates realistic synthetic data for 10 industrial assets with 72 hours of sensor readings. You can connect Supabase for persistent storage.
